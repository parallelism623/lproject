A generated value is the property where the value is expected to be generated by either:

* EF Core
* Database

The value is generated on insert and/or on update. In EF core, we can config it by multiple way:

* Data Annotations
* Fluent API: (Property -> map -> Database column)
  * *HasComputedColumnSql*: to specify that the property should map to a computed column
  * *HasDefaultValue*: to specify a default value that is used to pass to DB by EF core
  * *HasDefaultValue*: to specify the expression sql used to generate the default value.√
  * *ValueGeneratedOnAdd*: to specify that the selected property is generated by DB whenever</br>
    a new entity is added
  * *ValueGeneratedOnAddOrUpdate*: to specify that the selected property is generated whenever the entity</br>
    is added or modified. So it's useful to exclude the property from EF core's generated SQL.
    
The configuration type or value generated method (above) dictates how explicit values are handled</br>
on insert and update by determining the **PropertySaveBehavior** state, but does not explain value behavior:</br>
* **Save**: 
  * Insert without a generated value property, EF core will always insert provided value.
  * Insert  with a generated value property, EF core will always, insert provided value, </br>
  unless it's considered the default value of the property type (default value will not be sent to DB) 
  * Update: EF core will update the column if a value different form original (in DbContext instances) is provided</br>
* **Ignore**: 
  * Insert: EF core will always ignore the value.
  * Update: EF core will always ignore the value.
* **Throw**:
  * Insert: Ef core will throw an error if a value is provided
  * Update: EF core will throw an error if a value different from original is provided</br>

### Configuration Type – Default PropertySaveBehavior

| **Configuration Type** | **Behavior on Insert** | **Behavior on Update** |
|-------------------------|------------------------|------------------------|
| None                   | Save                   | Save                   |
| Computed               | Ignore                 | Ignore                 |
| Concurrency            | Save                   | Save                   |
| Default Value          | Save                   | Save                   |
| Key                    | Save                   | Throw                  |
| Identity               | Save                   | Save                   |
| Row Version            | Ignore                 | Ignore                 |

---

### ValueGenerated[XYZ] Methods – Default PropertySaveBehavior Mapping

| **ValueGenerated Method**          | **Behavior on Insert** | **Behavior on Update** |
|-----------------------------------|------------------------|------------------------|
| `ValueGeneratedNever`             | Save                   | Save                   |
| `ValueGeneratedOnAdd`             | Save                   | Save                   |
| `ValueGeneratedOnAddOrUpdate`     | Ignore                 | Ignore                 |
| `ValueGeneratedOnUpdate`          | Save                   | Ignore                 |
| `ValueGeneratedOnUpdateSometimes` | Save                   | Save                   |
